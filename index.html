GU3D DOCUMENTATION

Introduction

GU3D is a library to cretate 3D videogames for the PlayStation 2 exclusive on PlayLua (IDE), providing C/C ++ to programming. It is based on primitives extracted from open source projects, therefore I do not guarantee that it is free of Bugs.
It is important that you analyze the examples (PlayLua>Help>Examples>GU3D) so that you can understand how to use the library to program your game. If you have ever used OpenGL it will be easier for you.

Functionalities

  • 3D models: SMD (for now not include data animation)
  • Textures: BMP,JPG,PNG,TGA
  • Audio files: XM, WAV
  • Graphic primitives: points, lines and triangles
  • Ligths and shading
  • View(Camera) functions: First Person and Third Person
  • Math functions
  • Pad functions (only one)

Game assets

During the compilation of a program (main file), PlayLua (GU3D MODE) compiles into a file (Game Assets), all the textures and 3d models located in the "textures" and "models" folders respectively, which must be in the same folder as the main file. This means that loading a model or texture is done from memory and not from disk. Additionally within the respective folders, the textures and 3d models must not be in subfolders.

Audio files

Audio files do not yet fall into the Game Assets category, therefore these files are loaded from disk.

Graphics functions

Graphical primitives like matrix functions work similar to OpenGL with some restrictions. Matrix modes do not exist as in OpenGL, it can be said that it only works with the Modelview matrix. There is no orthogonal projection like in OpenGL, only work with perspective projection which can be adjusted with guProjection() a function similar to glFrustum. You can put the lights you want with guLight() although the cost will be seen in the shading calculations. Shading uses vertex coloring. The functions and the way of working with lights and shading is different from OpenGL, within a block limited by the calls to guBeginLights() and guEndLights() you can create the lights and draw the triangles or the models that will be affected by the lights , taking into account that it also has functions such as guMaterial() and guAmbient() where you can specify the effect of lights on drawings.

Structure of a program

#include <gu3d.h>
// Here includes, variables, functions, ...
int main(int argc, char * argv[]){
guInit();
// Here loading models, textures, sounds, music ...
while(1){
guBeginScene();
// Here loop of scene ...
guEndScene();
}
return(1);
}
